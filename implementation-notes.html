偏離情況

  • Norton SSL MITM hit npm install。第一次 npm install 出 UNABLE_TO_VERIFY_LEAF_SIGNATURE + npm bug「Exit handler never called」。Memory reference_norton_ssl_mitm.md 有寫 Python truststore fix 但 Node 冇 documented fix。揀 NODE_OPTIONS=--use-system-ca（Node 22+ flag，用 Windows cert store，bypass Norton MITM 嘅 cert verification 問題）— 12 秒 install 145 packages，clean exit。
  • 學員機可能撞同一 issue。HK 企業用 Norton / Symantec 比率高，pre-class checklist 要加一條「如 npm install 出 SSL error，跑 NODE_OPTIONS=--use-system-ca npm install」。

價值取捨

替代方案揀咗咩為何
npm config set strict-ssl falseNODE_OPTIONS=--use-system-castrict-ssl=false 全 disable cert verify = silent vulnerability，違反 Karpathy rule「fix root cause not symptom」。--use-system-ca 用 Windows cert store（含 Norton root） = 真正 fix，verify 仍 enforced。
